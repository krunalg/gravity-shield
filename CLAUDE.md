# Instructions for Claude Code

## Project Overview

Pi-hole AI Guardian: local DNS threat classification and threat-intel sync daemon for Raspberry Pi running Pi-hole and Ollama.

**Tech Stack:** Python 3.11, sqlite3, requests, watchdog, tldextract (offline PSL), dnspython, Ollama (local LLM, default `granite4.1:3b`), systemd

Dependencies are declared in `pyproject.toml` (`[project] dependencies`, dev tools in `[project.optional-dependencies]`). No requirements.txt.

**Key Locations:**
- Main code: `*.py` files in root
- Feature extraction: `features/*.py`
- Tests: `tests/test_*.py` (215 tests, all mocked, run with `make test`)
- Config: `config.py` defaults, `config_local.py` generated by setup
- Systemd: `pihole-ai.service.tpl`
- Build: `Makefile`

## Build & Test

```bash
make install
make test
make clean
```

## Architecture

**DomainWatcher** (`watcher.py`)
- Tails `/var/log/pihole/pihole.log`.
- Extracts new DNS query domains; skips domains covered by the never-block policy.
- TTL-aware seen-domain check via `StateDB.filter_unseen()` — re-classifies domains not seen in `SEEN_DOMAIN_TTL_DAYS`.
- Enqueues new domains to `queue.Queue(maxsize=500)` with `put_nowait()` — drops domain if queue is full.

**ClassifierWorker** (`watcher.py`)
- Consumes domains from queue one at a time (never parallel Ollama calls).
- Subdomain apex deduplication: if registered apex domain already blocked in StateDB, blocks subdomain directly without LLM call.
- Popularity allowlist: if apex domain is in the Tranco top list (rank ≤ `POPULARITY_RANK_THRESHOLD`) and not known to any threat feed, allows without LLM call. Threat-feed hits always override popularity.
- Rule pre-filter: skips LLM if `rule_score < RULE_PREFILTER_THRESHOLD` (default 15).
- Shared-hosting guard: subdomains on user-content platforms are never popularity-allowed; the provider name is passed through feature extraction (`SHARED_HOSTING_WEIGHT`) and LLM evidence, and RDAP age is skipped (provider age says nothing about the subdomain owner).
- Calls `DomainClassifier` for domains that pass pre-filter, passing the runtime brand map (see `brands.py`; cached `BRAND_MAP_REFRESH_SECONDS`), the RDAP domain age (see `rdap.py`; fail-open None on any error), and the ASN reputation (see `asn_reputation.py`; fail-open None).
- Blocks only when action/category and confidence pass thresholds AND the deterministic `rule_score >= BLOCK_RULE_SCORE_FLOOR` — the LLM-returned `risk_score` is advisory only.
- Logs classification and feature metadata to `StateDB`.

**ThreatIntelSyncer** (`syncer.py`)
- Fetches domain IOC feeds every 6 hours.
- Current default feeds: URLhaus and OpenPhish.
- Checks feed freshness at start of each cycle — warns if any feed not synced in `FEED_STALENESS_WARN_HOURS` (default 24h).
- Deduplicates known feed domains.
- Shared-hosting guard (`shared_hosting.py`): feed hostnames on user-content platforms (PSL private-domains section, synced weekly from publicsuffix.org into StateDB; offline fallback = tldextract bundled snapshot; `EXTRA_SHARED_HOSTING_SUFFIXES` seed) block the FULL hostname, bypassing popularity and rule-score thresholds — each subdomain has a distinct untrusted owner. The provider apex itself is never blocked.
- Popular-apex guard: never auto-blocks a non-shared-hosting domain whose apex holds a Tranco rank — feeds list URLs on compromised legitimate sites.
- Syncs the Spamhaus ASN-DROP list daily into StateDB `bad_asns` (used by the watcher, not for feed verification).
- Verifies remaining feed domains using **rule-based scoring only** (no Ollama). URLhaus domains receive `urlhaus_hit=True` → rule_score=100 → always pass. OpenPhish domains scored by lexical/entropy/TLD/DGA features.
- Ollama is not used for feed verification — feeds can have 40k+ entries, making LLM calls impractical.
- Per-feed try/except: one bad feed does not crash the sync cycle.
- TI block expiry: every fetched feed domain refreshes `last_seen` in StateDB `threat_domains`; after each sync cycle, `TI:` blocks not re-seen in any feed for `TI_BLOCK_EXPIRY_DAYS` (default 30, 0 = never) are removed from gravity.db via `PiholeClient.remove_from_denylist()` (comment-prefix guarded — never touches manual or `AI:` entries).
- Logs feed syncs and classification decisions.

**Brand Map** (`brands.py`)
- No hardcoded brand list. `get_brand_map(state_db)` derives `{brand_token: official_apex}` from the StateDB Tranco snapshot: apexes ranked ≤ `BRAND_SOURCE_RANK_THRESHOLD` (default 1000), token = SLD, tokens shorter than `BRAND_MIN_TOKEN_LENGTH` (default 5) dropped.
- `EXTRA_BRANDS` config seed adds/overrides entries for targets below the rank cutoff (regional banks etc.); it is also the fallback if the DB is unavailable.
- `features/brand.py` `detect(domain, brands=None)` defaults to `EXTRA_BRANDS`; callers (watcher, syncer, classifier) pass the derived map.

**RDAP Domain Age** (`rdap.py`)
- `get_domain_age_days(apex, state_db)` — registration age in days via RDAP; IANA bootstrap for server discovery (in-memory cache, 7d). Runs only post-prefilter in the watcher (low volume); never for feeds.
- StateDB `domain_registration` cache: successes forever, failures negative-cached `RDAP_NEGATIVE_CACHE_DAYS`.
- Scored in `features/rules.py` (`DOMAIN_AGE_NEW_DAYS`/`_WEIGHT`, `DOMAIN_AGE_RECENT_DAYS`/`_WEIGHT`); `domain_age_days` also goes to the LLM evidence. Unknown age = no signal. Disable with `RDAP_ENABLED = False`.

**Shared Hosting Detection** (`shared_hosting.py`)
- `get_shared_hosting_suffixes(state_db)` — shared-hosting/user-content suffix set: StateDB PSL snapshot (private-domains section of publicsuffix.org, synced weekly by the syncer) → tldextract bundled-snapshot fallback → merged with `EXTRA_SHARED_HOSTING_SUFFIXES`; in-memory cache `SHARED_HOSTING_REFRESH_SECONDS`.
- `shared_hosting_provider(hostname, suffixes)` — longest label-boundary suffix match, case-insensitive.
- `features/lexical.py` `hostname()` is PSL-private-aware: on shared hosts the user-controlled label is analyzed (`paypal-login.github.io` → `paypal-login`); brand detection scans both that label and the ICANN registrable label.

**ASN Reputation** (`asn_reputation.py`)
- Spamhaus ASN-DROP feed (JSONL) → StateDB `bad_asns`, synced daily by the syncer; empty fetch never wipes the list.
- `get_domain_asn(domain, state_db)` — A-record via `UPSTREAM_DNS_SERVER` directly (never through Pi-hole — loop), IP→ASN via Team Cymru DNS (`origin.asn.cymru.com` TXT). Cached in StateDB `domain_asn` for `ASN_CACHE_DAYS` (positives expire too — hosting moves). Watcher LLM path only; always fail-open.
- Flagged ASN adds `ASN_DROP_WEIGHT` to the rule score and `asn_flagged` to the LLM evidence. Disable with `ASN_REPUTATION_ENABLED = False`.

**TLS Certificate Analysis** (`tls_cert.py`) — OPT-IN, default off
- `TLS_ANALYSIS_ENABLED = False` by default: a live handshake connects to the suspected malicious host from this machine's IP.
- `get_cert_info(domain, state_db)` — stdlib `ssl` verified handshake (`TLS_TIMEOUT`); invalid cert (self-signed/expired/mismatch) is a signal (`TLS_INVALID_WEIGHT`), unreachable/no-TLS is no signal (None). Cert younger than `TLS_NEW_CERT_DAYS` adds `TLS_NEW_CERT_WEIGHT` (weak — free CAs rotate). Cached in StateDB `domain_tls` for `TLS_CACHE_DAYS`; `cert_age_days` recomputed from stored notBefore.
- Watcher LLM path only, fail-open; issuer/age/SAN count go to the LLM evidence.

**Popularity Trust List** (`popularity.py`)
- Fetches the Tranco top-1M list (zip or plain CSV), keeps domains ranked within `POPULARITY_RANK_THRESHOLD`.
- Synced by `ThreatIntelSyncer` every `POPULARITY_SYNC_INTERVAL_HOURS` (weekly); stored in StateDB `popular_domains` table, atomically replaced per sync.
- Empty/failed fetch never wipes the existing list.
- Generic replacement for hardcoded legit-domain lists: established domains skip the LLM entirely.

**Feature Extraction** (`features/`)
- Deterministically extracts lexical, entropy, digit, hyphen, punycode, TLD, brand, DGA, and rule-score evidence.
- Apex/hostname extraction uses the Public Suffix List via `tldextract` (offline bundled snapshot — `suffix_list_urls=()`), so multi-part TLDs like `.co.in`/`.co.uk` resolve correctly.
- Brand detection takes the brand map as a parameter (`detect(domain, brands=None)`) and returns `match_type`: `official` (registered domain is the brand's official apex in the brand map), `exact`, `contains`, `embedded` (hyphen-part), `leet` (character substitution), or `fuzzy`. Only non-`official` matches add rule score; the LLM prompt explains each type.
- `features.extractor.extract(domain, threat_context=None, brands=None)` is the only feature entry point used by `classifier.py`.
- `_build_evidence()` in `classifier.py` distils the full feature dict into a concise flat JSON before sending to the LLM — removes noisy nested lexical fields, keeps key signals only.
- The LLM receives structured evidence and should reason over it, not calculate features itself.

**Classifier** (`classifier.py`)
- Sends structured feature evidence JSON to the local LLM via Ollama.
- Expects JSON with `classification`, `confidence`, `severity`, `risk_score`, `reasons`, and `recommended_action`.
- Blocks only when action/category, confidence, and rule score pass configured thresholds.
- Only used by `watcher.py` (real-time DNS queries). `syncer.py` uses rule-based scoring instead.

**Pi-hole Writes** (`pihole_client.py`)
- Writes blocked domains to `gravity.db`.
- Creates/uses the Pi-hole group `Adaptive Threat Blocklist`.
- Maps all new blocked entries to that group via `domainlist_by_group`.
- Batches `pihole reloadlists` with `PIHOLE_RELOAD_INTERVAL_SECONDS`.
- Refuses to insert domains covered by the never-block policy, even if callers request it.

## Configuration

`config.py` contains defaults; `config_local.py` overrides local paths and runtime values.

Important settings:
- `OLLAMA_MODEL` (default: `granite4.1:3b`, any Ollama-compatible model)
- `BLOCK_CONFIDENCE_THRESHOLD`
- `RULE_SCORE_THRESHOLD` — syncer: minimum rule_score to block non-URLhaus feed domains
- `BLOCK_RULE_SCORE_FLOOR` — watcher: deterministic rule_score floor for LLM-driven blocks (default 15)
- `RULE_PREFILTER_THRESHOLD` — skip LLM if rule_score below this (default 15)
- `SEEN_DOMAIN_TTL_DAYS` — re-classify domains not seen in N days (default 7)
- `FEED_STALENESS_WARN_HOURS` — warn if feed not synced in N hours (default 24)
- `TI_BLOCK_EXPIRY_DAYS` — unblock TI: feed domains not re-seen in N days (default 30, 0 = never expire); expiry writes an `EXPIRED` verdict so subdomain dedup stops
- `DB_RETENTION_DAYS` / `DB_PRUNE_INTERVAL_HOURS` — StateDB history pruning (daily, default 90d)
- `PSL_FEED_URL` / `PSL_FEED_NAME` / `PSL_SYNC_INTERVAL_HOURS` — PSL private-domains sync (weekly)
- `EXTRA_SHARED_HOSTING_SUFFIXES` / `SHARED_HOSTING_REFRESH_SECONDS` / `SHARED_HOSTING_WEIGHT` — shared-hosting seed, cache TTL, rule weight
- `ASN_REPUTATION_ENABLED` / `ASN_DROP_FEED_URL` / `ASN_DROP_FEED_NAME` / `ASN_SYNC_INTERVAL_HOURS` — Spamhaus ASN-DROP sync
- `UPSTREAM_DNS_SERVER` / `ASN_LOOKUP_TIMEOUT` / `ASN_CACHE_DAYS` / `ASN_DROP_WEIGHT` — ASN lookups and scoring
- `TLS_ANALYSIS_ENABLED` (default False, opt-in) / `TLS_TIMEOUT` / `TLS_CACHE_DAYS` / `TLS_INVALID_WEIGHT` / `TLS_NEW_CERT_DAYS` / `TLS_NEW_CERT_WEIGHT` — TLS certificate analysis
- `POPULARITY_FEED_URL` / `POPULARITY_FEED_NAME` — Tranco top-1M list
- `POPULARITY_RANK_THRESHOLD` — apex domains ranked within this skip the LLM (default 100000)
- `POPULARITY_SYNC_INTERVAL_HOURS` — popularity list refresh interval (default 168 = weekly)
- `DGA_THRESHOLD`
- `ENTROPY_THRESHOLD`
- `BRAND_MATCH_THRESHOLD`
- `HIGH_RISK_TLDS`
- `EXTRA_BRANDS` — user brand seed below the Tranco rank cutoff
- `BRAND_SOURCE_RANK_THRESHOLD` / `BRAND_MIN_TOKEN_LENGTH` / `BRAND_MAP_REFRESH_SECONDS` — brand map derivation
- `NEVER_BLOCK_DOMAINS` / `NEVER_BLOCK_SUFFIXES` — infrastructure only; established domains are protected data-driven (popularity)
- `USER_ALLOWLIST_PATH` — user allowlist file (`domain.com` exact, `.domain.com` suffix), live-reloaded; false-positive recovery path
- `RDAP_ENABLED` / `RDAP_BOOTSTRAP_URL` / `RDAP_TIMEOUT` / `RDAP_NEGATIVE_CACHE_DAYS` — RDAP age lookups
- `DOMAIN_AGE_NEW_DAYS` / `DOMAIN_AGE_NEW_WEIGHT` / `DOMAIN_AGE_RECENT_DAYS` / `DOMAIN_AGE_RECENT_WEIGHT` — age scoring
- `THREAT_INTEL_FEEDS`
- `PIHOLE_BLOCK_GROUP_NAME`
- `PIHOLE_RELOAD_INTERVAL_SECONDS`

## Testing Strategy

All tests are mocked; no real Ollama, feed, or Pi-hole calls.

- `test_features.py` — deterministic feature extraction, PSL apex extraction, brand match_type provenance
- `test_classifier.py` — structured prompt and verdict parsing, brand match_type guidance in prompt
- `test_pihole_client.py` — gravity DB writes, never-block guard, group assignment, reload batching, comment-prefix-guarded removal, legacy block migration (`make migrate-blocks`)
- `test_syncer.py` — threat-intel rule-based verification, popular-apex feed guard, shared-hosting full-hostname blocking, feed freshness, feed error isolation, popularity/PSL/ASN-DROP sync scheduling, TI block expiry (last_seen touch, expiry cycle, disable with 0)
- `test_shared_hosting.py` — PSL private-section parsing, suffix matching, StateDB/snapshot/EXTRA fallback chain
- `test_asn_reputation.py` — ASN-DROP parsing, Team Cymru TXT parsing, ASN cache (positive/negative/stale), fail-open
- `test_tls_cert.py` — cert parsing, verify-failure signal, no-TLS no-signal, TLS cache (positive/negative/stale), fail-open
- `test_watcher.py` — watcher skip policy, queue enqueue/drop, ClassifierWorker (prefilter, popularity allowlist, subdomain dedup, classify-and-block)
- `test_state_db.py` — state schema/migration, TTL-aware seen-domain tracking, verdict lookup, sync hours, popularity rank storage
- `test_popularity.py` — Tranco list parsing (zip + plain CSV), rank threshold, fetch error handling
- `test_brands.py` — brand map derivation (SLD tokens, min length, rank cutoff, EXTRA_BRANDS override, DB fallback)
- `test_domain_policy.py` — user allowlist file (exact/suffix entries, comments, live reload, missing file)
- `test_rdap.py` — bootstrap parsing, registration event extraction, positive/negative caching, fail-open behavior
- `test_ollama_client.py` — Ollama HTTP wrapper
- `test_threat_intel.py` — feed parsing/fetch errors

## Deployment Notes

Setup generates:
- `config_local.py`
- `/etc/systemd/system/pihole-ai-$USER.service`
- `/etc/sudoers.d/pihole-ai-$USER`

The daemon runs as an unprivileged user. It needs write access to `/etc/pihole/gravity.db` and passwordless permission for the configured `pihole reloadlists` command.

## Debugging

```bash
make daemon-status
sudo journalctl -u pihole-ai-$USER -f
make logs
sqlite3 state.db "SELECT domain, category, confidence, rule_score, blocked FROM classifications ORDER BY classified_at DESC LIMIT 10"
sudo sqlite3 /etc/pihole/gravity.db "SELECT domain, comment FROM domainlist WHERE comment LIKE 'AI:%' OR comment LIKE 'TI:%' LIMIT 20"
sudo sqlite3 /etc/pihole/gravity.db "SELECT name FROM \"group\" WHERE name='Adaptive Threat Blocklist'"
```

## Constraints

**Do not:**
- Add cloud-based APIs or telemetry.
- Reintroduce retired/unresolvable feeds without verification and tests.
- Bypass `PiholeClient` for blocking.
- Insert into Pi-hole without the never-block guard.
- Hardcode user-specific paths or Pi-hole binary paths.

**Always:**
- Run `make test` before handoff.
- Update `config.py` when adding thresholds or tunables.
- Persist classification decisions through `StateDB`.
- Keep blocked entries in `Adaptive Threat Blocklist`.
- Keep Granite as the reasoning engine; deterministic modules own feature calculation.
