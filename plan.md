# Pi-hole AI Guardian — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Python daemon on any Raspberry Pi running Pi-hole that combines real-time AI-powered DNS domain classification (via local Ollama/Granite) with automated threat intelligence feed sync — both feeding directly into Pi-hole's denylist with zero cloud dependency.

**Architecture:** A single systemd-managed Python daemon with two parallel workers: (1) a `ThreatIntelSyncer` that fetches IOC feeds every 6 hours and bulk-inserts new malicious domains into Pi-hole's SQLite DB, and (2) a `DomainWatcher` that tails Pi-hole's FTL log in real-time, extracts domains never seen before, sends them to the local Ollama API for classification, and auto-blocks anything classified as malware/phishing/C2 with >80% confidence. Both workers share a lightweight SQLite state DB to avoid re-processing domains.

**Tech Stack:** Python 3.11, sqlite3 (stdlib), requests, watchdog (log tailing), Ollama REST API (localhost:11434), Pi-hole SQLite DB (/etc/pihole/gravity.db), systemd, granite3.3:2b model.

**Deploy path:** Generic, configured at install time via setup wizard — all usernames, IPs, paths parameterized into `config_local.py`.

## Pre-Task 0: Setup Wizard (Interactive Configuration)

**Files:**
- Create: `setup.sh` (interactive, prompts for all config)
- Generate: `config_local.py` (output from setup.sh)
- Generate: `/etc/systemd/system/pihole-ai-$USER.service` (from template)

**What this does:** User runs `./setup.sh` once. Script prompts for:
1. SSH username to use (default: current `$USER`)
2. Raspberry Pi IP or hostname running Pi-hole
3. Pi-hole admin password
4. Installation directory (default: `$HOME/pihole-ai`)
5. Ollama API URL (default: `http://localhost:11434`)

Then creates venv, installs deps, generates `config_local.py`, installs systemd service, starts daemon.

- [ ] **Step 1: Create setup.sh (interactive wizard)**

```bash
#!/usr/bin/env bash
set -e

echo "╔════════════════════════════════════════════╗"
echo "║  Pi-hole AI Guardian Setup Wizard          ║"
echo "╚════════════════════════════════════════════╝"
echo ""

# Prompt for configuration
read -p "SSH Username (current: $USER): " -r SSH_USER
SSH_USER=${SSH_USER:-$USER}

read -p "Pi-hole IP or hostname [192.168.68.104]: " -r PIHOLE_IP
PIHOLE_IP=${PIHOLE_IP:-192.168.68.104}

read -sp "Pi-hole Admin Password: " PIHOLE_PASSWORD
echo ""
read -sp "Pi-hole Admin Password (confirm): " PIHOLE_PASSWORD_CONFIRM
echo ""
if [[ "$PIHOLE_PASSWORD" != "$PIHOLE_PASSWORD_CONFIRM" ]]; then
    echo "ERROR: Passwords don't match"
    exit 1
fi

read -p "Installation directory [$HOME/pihole-ai]: " -r INSTALL_DIR
INSTALL_DIR=${INSTALL_DIR:-$HOME/pihole-ai}

read -p "Ollama API URL [http://localhost:11434]: " -r OLLAMA_URL
OLLAMA_URL=${OLLAMA_URL:-http://localhost:11434}

read -p "Ollama Model [granite3.3:2b]: " -r OLLAMA_MODEL
OLLAMA_MODEL=${OLLAMA_MODEL:-granite3.3:2b}

echo "Configuration Summary:"
echo "  SSH User: $SSH_USER"
echo "  Pi-hole IP: $PIHOLE_IP"
echo "  Install Dir: $INSTALL_DIR"
echo "  Ollama URL: $OLLAMA_URL"
echo "  Ollama Model: $OLLAMA_MODEL"

if [[ "$CONFIRM" != "y" ]]; then
    echo "Cancelled."
    exit 0

echo "Creating installation directory..."

source .venv/bin/activate

import subprocess

SSH_USER = "$SSH_USER"
PIHOLE_IP = "$PIHOLE_IP"
PIHOLE_PASSWORD = "$PIHOLE_PASSWORD"
INSTALL_DIR = "$INSTALL_DIR"
OLLAMA_URL = "$OLLAMA_URL"
OLLAMA_MODEL = "$OLLAMA_MODEL"

# Derived paths
BASE_DIR = INSTALL_DIR
STATE_DB_PATH = os.path.join(BASE_DIR, "state.db")
LOG_DIR = os.path.join(BASE_DIR, "logs")
PIHOLE_DB_PATH = "/etc/pihole/gravity.db"
FTL_LOG_PATH = "/var/log/pihole/pihole.log"

# Override config defaults
OLLAMA_BASE_URL = OLLAMA_URL

# Substitute variables
sed -i "s|\$SSH_USER|$SSH_USER|g" config_local.py
sed -i "s|\$PIHOLE_IP|$PIHOLE_IP|g" config_local.py
sed -i "s|\$PIHOLE_PASSWORD|$PIHOLE_PASSWORD|g" config_local.py
sed -i "s|\$INSTALL_DIR|$INSTALL_DIR|g" config_local.py
sed -i "s|\$OLLAMA_URL|$OLLAMA_URL|g" config_local.py
sed -i "s|\$OLLAMA_MODEL|$OLLAMA_MODEL|g" config_local.py

echo "✓ config_local.py created"

# Git init
if [[ ! -d .git ]]; then
    git init
    git config user.email "pihole-ai@localhost"
    git config user.name "Pi-hole AI Guardian"
    git add requirements.txt setup.sh
    git commit -m "Initial commit"
fi

echo ""
echo "✓ Setup complete!"
echo ""
echo "Next steps:"
echo "  1. Create project files (Tasks 1-10)"
echo "  2. Run tests: pytest tests/ -v"
echo "  3. Install systemd service: sudo systemctl enable pihole-ai-$SSH_USER"
```

- [ ] **Step 1: Create pihole-ai.service.tpl**

```ini
[Unit]
Description=Pi-hole AI Guardian — AI domain classifier + threat intel sync
After=network-online.target pihole-FTL.service ollama.service
Wants=network-online.target
Requires=pihole-FTL.service

[Service]
Type=simple
User=$SSH_USER
WorkingDirectory=$INSTALL_DIR
ExecStart=$INSTALL_DIR/.venv/bin/python daemon.py
Restart=on-failure
RestartSec=30
StandardOutput=journal
StandardError=journal
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

- [ ] **Step 2: Update setup.sh to generate systemd unit**

Add to `setup.sh` after `config_local.py` generation:

```bash
# Generate systemd service from template
echo "Generating systemd service..."
sed -e "s|\$SSH_USER|$SSH_USER|g" \
    -e "s|\$INSTALL_DIR|$INSTALL_DIR|g" \
    pihole-ai.service.tpl > pihole-ai-$SSH_USER.service

echo "Installing systemd service..."
sudo cp pihole-ai-$SSH_USER.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable pihole-ai-$SSH_USER

# Allow user to write to gravity.db and run reloadlists
SUDOERS_LINE="$SSH_USER ALL=(ALL) NOPASSWD: /usr/bin/pihole reloadlists, /usr/bin/pihole reloaddns"
if ! sudo grep -qF "pihole reloadlists" /etc/sudoers.d/pihole-ai-$SSH_USER 2>/dev/null; then
    echo "$SUDOERS_LINE" | sudo tee /etc/sudoers.d/pihole-ai-$SSH_USER > /dev/null
    sudo chmod 0440 /etc/sudoers.d/pihole-ai-$SSH_USER
    echo "✓ Sudoers rule created"
fi

# Start daemon
echo "Starting daemon..."
sudo systemctl start pihole-ai-$SSH_USER
```
